# Получение Client ID / Client Secret для BOG Business (веб-версия)

## Шаг 1. Войдите в BOG Business (веб-версия)

![Скриншот шага 1](https://taxflowge.github.io/images/bog_business_creds_step_1.png)

Откройте сайт **bonline.bog.ge** и переключитесь в режим **BUSINESS** (верхнее меню).

**Что нужно сделать:**
- введите **Username** и **Password** от вашего **BOG Business** аккаунта;
- нажмите **LOG IN**.

Также можно войти через QR-код: отсканируйте его в приложении **Business Mobile Bank**.

**Результат шага:** вы **успешно авторизовались** в **BOG Business (веб-версия)** и переходите к подтверждению входа.

---

## Шаг 2. Подтвердите вход одноразовым кодом

![Скриншот шага 2](https://taxflowge.github.io/images/bog_business_creds_step_2.png)

После ввода логина и пароля банк отправит **одноразовый код** для подтверждения входа.

**Что нужно сделать:**
- дождитесь код (он действует ограниченное время);
- введите его в поле **Enter code**;
- нажмите **NEXT**.

**Результат шага:** вход в **BOG Business (веб-версия)** подтверждён, вы переходите в аккаунт.

---

## Шаг 3. Перейдите в раздел API Integration

![Скриншот шага 3](https://taxflowge.github.io/images/bog_business_creds_step_3.png)

После входа в интернет-банк откройте раздел **API Integration** — именно там создаются ключи для подключения банка к TaxFlow.

**Что нужно сделать:**
- в верхнем меню нажмите **API Integration**.

**Примечание:** *на скриншоте скрыты чувствительные данные (счета/балансы/идентификаторы) — у вас будут отображаться ваши реальные значения.*

**Результат шага:** вы открыли раздел **API Integration** и готовы создать **Client ID / Client Secret**.

---

## Шаг 4. Создайте API Client

![Скриншот шага 4](https://taxflowge.github.io/images/bog_business_creds_step_4.png)

В разделе **API Integration** нужно создать клиента, который выдаст ключи для подключения банка к TaxFlow.

**Что нужно сделать:**
- в блоке **Add an API Client** нажмите кнопку **ADD**.

**Примечание:** *на скриншоте скрыты чувствительные данные (идентификаторы/аккаунт) — у вас будут отображаться ваши реальные значения.*

**Результат шага:** вы перешли к созданию **API Client**.

---

## Шаг 5. Выберите тип подключения Client Credentials Flow

![Скриншот шага 5](https://taxflowge.github.io/images/bog_business_creds_step_5.png)

Откроется окно **ADDING CLIENT** с вариантами типа интеграции. Для TaxFlow нужен тип, который выдаёт **Client ID / Client Secret**.

**Что нужно сделать:**
- выберите **Client Credentials Flow**;
- нажмите **NEXT**.

**Результат шага:** выбран корректный тип интеграции, вы переходите к созданию клиента и получению ключей.

---

## Шаг 6. Укажите имя клиента и выберите пользователя

![Скриншот шага 6](https://taxflowge.github.io/images/bog_business_creds_step_6.png)

После выбора **Client Credentials Flow** нужно заполнить данные клиента, от имени которого будут выданы ключи.

**Что нужно сделать:**
- в поле **API Client name** введите любое понятное имя (например, **TaxFlow**);
- в поле **Internetbank user** выберите вашего пользователя из списка;
- нажмите **NEXT**.

**Примечание:** *на скриншоте скрыты чувствительные данные (аккаунт/пользователь) — у вас будут отображаться ваши реальные значения.*

**Результат шага:** клиент создан с нужным пользователем, вы переходите к подтверждению создания.

---

## Шаг 7. Подтвердите создание клиента (SMS или QR)

![Скриншот шага 7](https://taxflowge.github.io/images/bog_business_creds_step_7.png)

После нажатия **NEXT** банк попросит подтвердить действие одноразовым кодом.

**Что нужно сделать:**
- выберите способ подтверждения (**SMS** или **QR code**);
- введите **One time code** (если выбрали SMS);
- нажмите **CONFIRM**.

**Результат шага:** создание **API Client** подтверждено, вы переходите к экрану с ключами.

---

## Шаг 8. Скопируйте Client ID и Client Secret

![Скриншот шага 8](https://taxflowge.github.io/images/bog_business_creds_step_8.png)

После подтверждения появится экран **API Client has been added**. Здесь банк показывает ключи, которые нужны для подключения **BOG Business** в TaxFlow.

**Что нужно сделать:**
- скопируйте **Client ID**;
- скопируйте **Client Secret**;
- сохраните их в надежном месте — они понадобятся на шаге подключения банка в TaxFlow.

**Примечание:** *на скриншоте показаны примерные значения. Не передавайте **Client Secret** третьим лицам.*

**Результат шага:** у вас есть **Client ID / Client Secret** для ввода в TaxFlow.

---

## Шаг 9. Выдайте права API: National Currency Transfer

![Скриншот шага 9](https://taxflowge.github.io/images/bog_business_creds_step_9.png)

После создания клиента нужно выдать ему права — без этого TaxFlow не сможет создавать платежный документ для оплаты налога.

**Что нужно сделать:**
- откройте вкладку **API PERMISSIONS**;
- в блоке **ACCOUNT** выберите пункт **Current account in base currency**;
- в списке прав отметьте **National Currency Transfer**.

**Примечание:** *на скриншоте скрыты чувствительные данные (ID, контакты, счета) — у вас будут отображаться ваши реальные значения.*

**Результат шага:** выдали нужное разрешение, TaxFlow сможет сформировать платежный документ для оплаты налога.

---

## Шаг 10. В Transfers and Payments включите View Details для Between my accounts (для каждой валюты)

![Скриншот шага 10](https://taxflowge.github.io/images/bog_business_creds_step_10.png)

На странице **API PERMISSIONS** мы настраиваем права **на просмотр транзакций** — без этого TaxFlow не сможет получить операции по счетам.

**Что нужно сделать:**
- в разделе **Transfers and Payments** выберите пункт **Between my accounts**;
- для **каждого счета в каждой валюте** включите право **View Details**.

**Примечание:** *на скриншоте скрыты чувствительные данные (номера счетов и персональные данные) — у вас будут отображаться ваши реальные значения.*

**Результат шага:** права на просмотр транзакций включены, TaxFlow сможет подтянуть операции по счетам.

---

## Шаг 14. Примените изменения прав и подтвердите кодом

![Скриншот шага 14](https://taxflowge.github.io/images/bog_business_creds_step_14.png)

После того как вы выставили права (**View Details** и **National Currency Transfer**), нужно применить изменения — иначе они не сохранятся.

**Что нужно сделать:**
- внизу страницы нажмите кнопку **Update** (применить изменения);
- выберите способ подтверждения (**SMS** или **QR code**);
- введите **One time code** (если выбрали SMS);
- нажмите **CONFIRM**.

**Результат шага:** права сохранены и активированы — TaxFlow сможет получать транзакции и формировать платежный документ для оплаты налога.

---

## Шаг 15. Убедитесь, что права обновлены

![Скриншот шага 15](https://taxflowge.github.io/images/bog_business_creds_step_15.png)

После подтверждения появится сообщение **Permissions updated successfully** — это означает, что изменения прав сохранены.

**Что нужно сделать:**
- нажмите **CLOSE**.

**Результат шага:** права применены и активны — можно возвращаться в TaxFlow и подключать **BOG Business** с вашими **Client ID / Client Secret**.
